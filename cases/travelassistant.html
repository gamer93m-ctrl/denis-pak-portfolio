Overview

Travel Assistant is a mobile app that consolidates flight booking, hotel reservations, and event discovery into a single seamless experience. Users can plan and book an entire trip without switching between multiple apps.

My responsibility covered the end-to-end UX and UI design: from user research through final handoff — including the smart form auto-fill system and camera-based document scanning for profile setup.

App overview — key screens

Research & Methodology

I interviewed 12 frequent travellers to understand where existing travel apps create the most friction. The dominant pain point: switching between 3–5 apps to complete a single trip, with repetitive data entry at each step.

Two key insights shaped the design direction:

  • Users want smart defaults — the app should remember their preferences and pre-fill forms
  • Document scanning removes the biggest onboarding barrier (manually entering passport details)

Design Challenges & Solutions

Challenge 1 — Unified booking flow

Combining flights, hotels, and events in one interface risks creating a cluttered, confusing navigation. I solved this with a clear bottom-tab structure and a unified search layer that adapts its form fields based on the selected travel category.

Booking flow — flight, hotel, event screens

Challenge 2 — Smart form auto-fill

Repetitive data entry is the top abandonment trigger in travel booking. I designed a persistent profile layer that pre-fills passenger details, payment info, and preferences across all booking types — reducing form completion time by surfacing only fields that differ from saved defaults.

Empty form state
Auto-filled form state

Challenge 3 — Camera document scanning

For first-time users, setting up a travel profile requires entering passport and ID data. I designed a camera-based scanning flow using the device camera to extract key fields automatically, with a simple correction step for any mis-reads — reducing onboarding drop-off significantly.
