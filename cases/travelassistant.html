Приложение «всё-в-одном» для бронирования авиабилетов, отелей и мероприятий. Запуск с сильными метриками в первый месяц и рейтингом 4.2★.

Обзор

Travel Assistant — мобильное приложение, объединяющее бронирование авиабилетов, отелей и мероприятий в одном интерфейсе. Пользователь может спланировать и забронировать всю поездку, не переключаясь между приложениями.

Моя ответственность охватывала весь UX и UI дизайн: от исследования пользователей до финальной передачи в разработку — включая систему автозаполнения форм и сканирование документов камерой для настройки профиля.

Исследование и методология

Провёл интервью с 12 частыми путешественниками, чтобы понять, где существующие travel-приложения создают наибольшее трение. Главная боль: переключение между 3–5 приложениями для оформления одной поездки с повторным вводом данных на каждом шаге.

Два ключевых инсайта определили дизайн-направление:

  • Пользователи хотят умных значений по умолчанию — приложение должно помнить предпочтения и заполнять формы
  • Сканирование документов снимает главный барьер онбординга (ручной ввод паспортных данных)

Дизайн-задачи и решения

Задача 1 — Единый флоу бронирования

Объединение авиабилетов, отелей и мероприятий в одном интерфейсе рискует создать перегруженную навигацию. Решил с помощью чёткой нижней навигации и единого поискового слоя, который адаптирует поля формы под выбранную категорию.

Флоу бронирования — авиа, отели, мероприятия

Задача 2 — Умное автозаполнение форм

Повторный ввод данных — главная причина отказов в travel-бронировании. Спроектировал постоянный профильный слой, который предзаполняет данные пассажира, платёжную информацию и предпочтения для всех типов бронирования.

Задача 3 — Сканирование документов камерой

Для новых пользователей настройка travel-профиля требует ввода паспортных данных. Спроектировал флоу сканирования камерой для автоматического извлечения данных с простым шагом коррекции — значительно снизив отток на онбординге.
